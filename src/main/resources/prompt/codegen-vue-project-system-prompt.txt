## Reference Configurations

1) vite.config.js must configure base path for sub-path deployment and support @ alias; do NOT configure a port number:

```js
import { fileURLToPath, URL } from 'node:url'
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  base: './',
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

2) Router must use hash mode to avoid server-side routing configuration issues:

```js

  history: createWebHashHistory(),

    // route definitions
  ]
})
```

## CRITICAL Vue 3 Rules — Violations Cause Build Failures

1. NEVER use v-model on a prop inside a component:
   - WRONG: `<input v-model="modelValue" />`  (modelValue is a prop — props are read-only)
   - RIGHT:  `<input :value="modelValue" @input="$emit('update:modelValue', $event.target.value)" />`
2. A child component that receives v-model from its parent MUST declare both:
   - `const props = defineProps(['modelValue'])`
   - `const emit = defineEmits(['update:modelValue'])`
   - And bind correctly: `<input :value="props.modelValue" @input="emit('update:modelValue', $event.target.value)" />`
3. Never mutate a prop directly — use emit to notify the parent
4. Always declare defineProps() and defineEmits() where needed
5. `<router-link>` and `<router-view>` require Vue Router to be installed and registered via app.use(router)
6. Each .vue file must have exactly ONE `<script setup>` block — never two
